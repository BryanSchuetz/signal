Syncing NewsFire

Feb 15 2009Tags: os x hacks

I just switched all my feed reading over to the newly released 1.6 version of NewsFire and I’m loving it. The interface is gorgeous and the keyboard shortcuts make powering through a large number of feeds a breeze. Add in the new twitter plugin for Safari, subscribe to some key twitter users and you’ve almost rolled your own twitter client.

There was one problem though, no syncing. If I’m at work paging through some new items, when I get home I want those items to show up as having been read. The solution turned out to be shockingly simple: download the free version of Dropbox create some symbolic links to a few key files on each computer and it’s done. Changes made on one computer are synced through Dropbox and show up on the other.

Just move this folder:

~/library/application support/newsfire

And this file:

~/Library/preferences/org.xlife.NewsFire.plist

into the DropBox and replace them with symbolic links. To create a symbolic link just fire up the terminal and use the ln command. The format will be (ln -s) (filepath to target) (filepath for link), for example:

ln -s /Users/yournamehere/Dropbox/newsfiresymlinks/org.xlife.NewsFire.plist /Users/yournamehere/Library/Preferences

It really is that simple and from what I can see, works like a charm.

Update:Depending on the privileges you have when creating the symlink you may need to lock the file after you create it so NewsFire does not over write it when you exit.
